📖 1. Official Microsoft Learn Modules

These six modules form the official learning path for GH-600. Every exam question maps to content in these modules. Start here.

Foundations of Agentic AI in GitHub

Module All domains

Introductory module covering what agentic AI means in the GitHub ecosystem. Defines core concepts: agents, tools, MCP, Copilot, and the GitHub control plane. Prerequisite reading before diving into individual domains.

Good for: Building foundational vocabulary and understanding the big picture before studying individual domains.

aka.ms/AA11551d

Designing Agent Architecture and SDLC (Software Development Lifecycle) Integration

Module Domain 1

Deep dive into agent architecture patterns, SDLC integration points, planning vs execution boundaries, and observability setup. Maps directly to Domain 1 of the exam.

Good for: Understanding agent architecture patterns, SDLC integration, and planning/execution separation.

aka.ms/AA11551g

Tooling, MCP, and Agent Execution Environments

Module Domain 2

Covers the full tool ecosystem: built-in tools, MCP servers, CI integration, safe execution paths, and error handling. This is the heaviest domain (20-25%).

Good for: Everything tool-related — tool selection, MCP configuration, execution environments, error handling.

aka.ms/AA11551m

Memory, State, and Evaluation

Module Domains 3 & 4

Two domains in one module: managing agent memory/state (Domain 3) and performing evaluation, error analysis, and tuning (Domain 4). Covers Copilot Memory, context management, and the PAE loop.

Good for: Understanding memory strategies, state management, evaluation criteria, and tuning levers.

aka.ms/AA114xam

Multi-Agent Systems and Orchestration

Module Domain 5

Everything about multi-agent coordination: SDLC role separation, orchestration patterns (hierarchical, sequential, parallel, competitive), agent isolation, conflict detection, and failure recovery at scale.

Good for: Multi-agent design patterns, orchestration strategies, isolation mechanisms, and system observability.

aka.ms/AA114xao

Governance, Guardrails, and Operations

Module Domain 6

Risk-based autonomy, GitHub-native controls (rulesets, checks, CODEOWNERS, environments), human-in-the-loop workflows, least-privilege permissions, and governance maintenance. The safety and control module.

Good for: Understanding governance frameworks, guardrail configuration, human-in-the-loop design, and auditability.

aka.ms/AA114xaq

Exam Tips from Ari Levigny (Microsoft Reactor)

Key takeaways from Ari Levigny's (GitHub Sr. Developer Advocate) Microsoft Reactor sessions. These are direct quotes and emphasis points from the videos — useful for last-minute review and study prioritization.

  • "Trust but verify" — evaluate work by workflow standards, not novelty of author
  • Plan → Act → Evaluate is the central cycle
  • Domain 2 (Tooling) is the heaviest — spend the most study time here
  • Planning agents should be read-only (just read/search tools)
  • Context rot/drift happens when context windows get too full
  • Agents never merge directly to main — always PR + checks + review
  • Execution without validation is an anti-pattern
  • Understand least privilege for tools, branches, environments
  • Beta exam counts — no retake needed at GA
  • GitHub Actions knowledge needed at high level only, not deep YAML
