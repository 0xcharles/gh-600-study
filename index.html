About This Exam

The GH-600: Developing in Agentic AI Systems certification validates your ability to operate, integrate, supervise, and govern AI agents inside production-grade SDLC (Software Development Lifecycle) workflows using GitHub as the system of record and control plane.

Unlike feature-based certifications, GH-600 is role-based and scenario-driven. You won't be tested on memorizing documentation — you'll be asked to make architectural decisions about agent systems.

Microsoft Reactor Deep Dive — Key Takeaways

Based on Ari Levigny's (GitHub Sr. Developer Advocate) 1-hour exam walkthrough. These are direct quotes and emphasis points from the video.

Core mantra — "Trust but verify": "Evaluate the work by the standards of the workflow, not by the novelty of the author. The human and the policy accept the agent's proposals." Agents propose, humans and rulesets decide what gets merged.

Plan → Act → Evaluate is the heartbeat: Plan your architecture, let agents act within scoped tools, then evaluate via checks and scans. Feed the evidence back into the plan. It's a cycle, not a straight line.

Domain 2 is the bulk of the exam: Ari explicitly says "spend a lot of time on the Tooling module." Planning agents should be read-only (just read/search). Execution agents (coder, designer) get edit/execute. Always ask: what is the least privilege needed?

Context rot / context drift is tested: When the context window gets too full, the model struggles and drifts from the main goal. Domain 3 tests your ability to manage memory expiration, pruning, and session resumption.

Multi-agent orchestration is a skill to learn for real work: "Not just for the exam." Orchestrator → Planner → Coder + Designer working in parallel on isolated branches. Each agent gets its own branch, proposes a PR, runs checks. Never merge to main directly.

Governance = least privilege + rulesets + CODEOWNERS + environment gates: Ari emphasizes "you got to understand what does least privilege mean." The agent operates in isolated branches with tightly scoped workflow permissions, branch policies, and rule sets. Required scans (CodeQL), human-in-the-loop via CODEOWNERS, and deployment approvals.

Execution without validation = anti-pattern: "If an agent can bypass required checks or merge without reviews, the architecture loses its primary safety mechanisms." Always keep human review + status checks + rulesets between the agent and main.

Actions knowledge needed — but not deep YAML: You need to understand how agents touch GitHub Actions (evaluating workflows, fixing errors in workflows). You don't need to know every YAML key or action detail. High-level understanding is enough.

Beta exam counts — no retake needed at GA: If you pass the beta, you're grandfathered in. The beta is for Microsoft to collect feedback on ambiguous questions. Results take ~8 weeks. Certification valid ~2-3 years.

Hands-on practice is how you learn: "Reading and videos are great, but hands-on practice really drives the point home." Do the GitHub Skills, run the Microsoft Learn modules, and use the exam sandbox (aka.ms/examdemo).
